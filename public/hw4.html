HW 4

It is time to implement the soccer team management application we speced in HW1 and prototyped in HW3. Traditionally as coding focused computing professionals we would spend a great deal of time building data structures and models first, but so far in this class we have designed from the interface down. In this step we should work upwards from the data and join the two halves. In this assignment we will build a partially functioning application and when finished all that be missing will be a few server-side pieces of the application.

Planning Document

Our group fleshed out a decent amount of features in this iteration of the homework. Using local storage, we were able to implement CRUD with the players, games, and game events. We were also able to handle log in, supporting multiple accounts, with the use of state management. In addition, we also added the capability to upload and view images, which were used for the team logo and the players' image.

Moving forward, we still have several features we would like to add. As it is required, we would like to actually hook up our site to an actual online backend, such as Firebase, and not have to use local storage. We would also like to add the ability to manage one's account. In addition, users can currently only sign up as coaches who manage teams. In the future, we would like to add the ability for players/parents/spectators to be able to view the team data, but not have the ability to modify it. Lastly, we would like to add some sort of mechanism for linking teams together in a league. Currently, users must manually add the other teams, which they cannot edit at the moment. This is just a temporary fix, in which we will fix fully later.
